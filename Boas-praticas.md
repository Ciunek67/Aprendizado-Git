## Boas Praticas 

1 - Escreva mensagens de commit claras e descritivas
Cada commit deve explicar de forma objetiva o que foi alterado. Evite mensagens genéricas como "update" ou "mudanças". Prefira algo como:

- Adiciona tela de login
- Corrige erro na validação de senha
- Atualiza documentação do projeto

2 - Utilize branches para desenvolver novas funcionalidades
Em vez de trabalhar diretamente na branch main, crie uma branch específica para cada funcionalidade ou correção. Isso facilita os testes, a revisão de código e evita problemas na versão principal do projeto.

3 - Faça commits pequenos e frequentes
Procure registrar mudanças relacionadas em commits separados, em vez de acumular muitas alterações em um único commit. Isso torna o histórico mais organizado e facilita encontrar ou reverter mudanças quando necessário.
